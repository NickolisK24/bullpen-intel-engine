# Bullpen Context Layers V1 Audit

Branch: `audit/context-layers-v1`

Scope: backend context-readiness audit for `rotation_context`, `bullpen_concentration_context`, `bullpen_optionality_context`, `role_stability_context`, `injury_context`, and aggregate `bullpen_context`.

## Verdict

The five context layers are ready to serve as neutral inputs for Story Intelligence V1.

They are not story generators. They do not create public UI, rankings, predictions, closer/setup classifications, leverage claims, quality claims, or scoring changes.

## Verified Boundaries

- `rotation_context` uses recorded starter/relief game-log splits and keeps bullpen innings out of starter averages.
- `bullpen_concentration_context` and `role_stability_context` share the same valid pitch-count workload authority for bullpen workload cores.
- `bullpen_optionality_context` uses current availability records and valid workload appearances; 0 IP / 0 pitch rows do not drive workload context.
- `injury_context` uses current roster/status data and current availability population counts; active relievers are not counted as inactive depth.
- Aggregate `bullpen_context` includes all five layers with stable no-data shapes and no story output.
- When no explicit reference date is provided, aggregate `bullpen_context` anchors to the latest stored team game-log date as its data-through date rather than browser-local or server UTC time.

## Known Limitations

- Game logs do not yet store team-at-appearance, so context layers use currently assigned pitchers.
- Opener and bulk-reliever roles are not separately inferred; credited starter usage is handled as recorded.
- Unknown `gamesStarted` rows are excluded from starter/reliever-specific workload context.
- Unknown or incomplete roster status lowers confidence or returns safe neutral injury-context values.
- If aggregate `bullpen_context` has no game-log reference date, it returns no-data defaults for all layers rather than manufacturing a context date.
- Callers that already have a platform freshness date should pass that date explicitly; this audit did not introduce a new freshness service dependency.

## Story Intelligence Readiness

The audit tests verify that one aggregate team context payload contains neutral inputs that can support future story structure slots:

- observation
- baseline
- cause
- interpretation
- forward constraint

No story text is generated by this audit.
