# Phase 0E Editorial Review Guide

## Packet Checklist

For every sampled read, inspect:

- read type, subject type, subject id, and product date;
- read completeness state;
- every component state;
- required versus optional component placement;
- every cited evidence id and quoted rendered claim;
- every reason code;
- every limitation;
- unknown, conflict, and withheld explanations;
- contributor denominator disclaimer where contributor composition appears;
- mandated limitation text for entry base state, HBP/ROE scope, schedule-change
  scope, appearance-count scope, and active-pitcher census scope.

Every sampled read must satisfy every checklist item. One failed checklist item
fails the packet.

## Overclaiming Spotting Guide

Watch for words that convert evidence completeness into readiness, use-state,
health, role, quality, depth, intent, pressure, or prediction. A complete read
means the required evidence components are complete; it does not mean the
subject is prepared, usable, preferred, better, deeper, or likely to be used.

```FORBIDDEN-PATTERN EXAMPLE
Available because the read is complete.
```

```FORBIDDEN-PATTERN EXAMPLE
Fresh enough for a late inning.
```

```FORBIDDEN-PATTERN EXAMPLE
The team is deep because contributor composition is complete.
```

```FORBIDDEN-PATTERN EXAMPLE
The pitcher will be used because the component has no conflict.
```

## Audit-Reading Discipline

Divergence is not a verdict. The legacy capture and the composed read are shown
beside each other for internal review. Unknown-driven divergence does not make a
legacy label wrong; it means the read does not adjudicate the captured display.

Material rows are recommendations for Nickolis to consider for legacy-engine
defect review. They do not create automatic fixes, freezes, public copy
changes, or payload changes.
