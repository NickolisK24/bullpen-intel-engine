# BaseballOS Narrative Governance

BaseballOS writing standards are governed by
[`docs/product/BASEBALLOS_WRITING_RULES.md`](../product/BASEBALLOS_WRITING_RULES.md).
That document is the source of truth for all user-facing BaseballOS language.

The standard applies across:

* Dashboard language
* Today page language
* Stories feed language
* Team pages
* Player and reliever summaries
* Trust and methodology explanations
* Social-ready content
* Automated or template-produced copy
* Any explanatory text shown to users

UX copy, generated stories, team summaries, daily observations, social-ready
copy, and explanatory text should comply with the writing rules before reaching
users.

Internal metric names and product-system labels should be translated into
baseball meaning before they appear in user-facing copy. Users should see the
baseball reality, not the internal machinery.

This document does not authorize new runtime behavior, story generation logic,
ranking behavior, selection behavior, prediction behavior, or recommendation
behavior. It only points future UX and story work to the narrative-language
standard.
